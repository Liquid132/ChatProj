### asio网络库

一个跨平台C++库，用于实现异步I/O。其采用Proactor模式，通过异步操作和时间循环，使用少量线程处理大量链接，减少阻塞和线程切换成本，从而实现高并发高性能
- 异步操作：用户发起网络请求后，asio不阻塞进程程等待数据返回，而是继续处理其他任务
- 事件循环：一个或几个 `io_context` 线程会负责监听操作系统（Linux下用 epoll）的通知。当内核汇报“数据准备好了”，ASIO 就会回头调用用户的回调函数来处理

**组件**
1. `io_context` 与操作系统交互，负责调度所有异步任务。使用`io_context.run()`启动事件循环
2. `socket` 网络通信接口，对应TCP、UDP等协议的通信端点
3. `timer` 定时器，用于处理超时或执行周期性任务
4. `strand` 线程安全绳。多线程环境下调用同一个`socket`的方法，`strand`保证回调函数不并发执行，避免数据竞争

`async`(`asynchronous, /āˈsiNGkrənəs/`)异步，在asio中是一种编程模式

异步操作不阻塞线程，线程在等待I/O时可以处理其余任务，具有高并发和资源效率高的优势

**asio异步编程流程**
```txt
1. 创建 io_context（事件循环引擎）
         ↓
2. 发起异步操作（如 async_read/async_write），并传入回调函数
         ↓
3. 调用 io_context.run() 启动事件循环
         ↓
4. 当操作完成时，ASIO 自动调用你的回调函数
         ↓
5. 在回调中可以再次发起异步操作，形成链式调用
```
- 所有异步操作依附`io_context`，必须调用`run()`
- 回调函数中如果有耗时操作，会阻塞整个事件循环
- 异步操作中`this`指针可能失效，一般使用`shared_ptr`和`enable_shared_from_this`

**`Proactor`和`Reactor`**
| 对比维度 | `Reactor`(反应器) | `Proactor`(前摄器) |
| :-- | :-- | :-- |
| 工作方式 | 同步非阻塞 | 异步 |
| 内核行为 | 通知数据准备完毕 | 通知数据处理完毕 |
| 程序行为 | 收到通知后调用`read`，将数据从内核拷贝到用户空间 | 收到通知时，数据已存在于用户空间，直接使用 |
| 回调时机 | I/O就绪回调 | I/O完成时回调 |
| 对应网络库 | libevent，libev，Redis，Nginx | ASIO（Windows上），Boost.Asio |

asio在Windows上使用Proactor，在Linux上，由于异步I/O接口不完善，通常使用Reactor（epoll）模拟Proactor

**Reactor模式**
```txt
应用程序                内核
   |                    |
   | 1. 注册可读事件      |
   |------------------->|
   |                    | (数据还没到)
   |                    |
   | 2. 通知“数据准备好了”|
   |<-------------------|
   |                    |
   | 3. 调用 read 读取   |
   |------------------->|
   |                    | 4. 内核拷贝数据到用户区
   |                    |    (阻塞等待拷贝完成)
   | 5. read 返回       |
   |<-------------------|
   |                    |
   | 6. 处理数据         |
```

**Proactor模式**
```txt
应用程序                内核
   |                    |
   | 1. 发起异步 read    |
   | (指定缓冲区地址)    |
   |------------------->|
   | 2. 立即返回         |
   |<-------------------|
   |                    | 3. 内核等待数据到达
   |                    | 4. 内核拷贝数据到用户缓冲区
   |                    |    (不占用应用程序)
   | 5. 通知“完成”       |
   |<-------------------|
   |                    |
   | 6. 直接使用缓冲区数据|
```
